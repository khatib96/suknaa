# UX / Product Backlog

> ملف دائم لملاحظات المنتج وتجربة المستخدم والمسميات. هذه الملاحظات ليست blockers على Phase 2 M9 إلا إذا نُقلت صراحة إلى milestone لاحق.

## 2026-05-07 — ملاحظات بعد Smoke Test لـ M9

- واجهة KYC لازم تصير حسب نوع المضيف. الحساب الشخصي لا يجب أن يرى حقول الشركات أو الفنادق غير المطلوبة له.
- مراجعة هل صورة السيلفي مع الهوية مطلوبة فعلاً للحساب الشخصي. إذا بقيت مطلوبة، يجب شرح السبب بوضوح في الواجهة.
- تعديل مسميات نوع الحساب:
  - `individual` -> `شخصي`
  - مسارات الشركات/المكاتب -> `تجاري`
- فصل تسجيل الفنادق عن تسجيل بيوت العطلات. onboarding الفنادق يجب أن يظهر كمسار hospitality/company مستقل.
- تعديل التصنيف العام في الواجهة عند اللزوم:
  - استبدال `عقارات` بـ `بيوت عطلات`
  - إبقاء مسميات الفنادق منفصلة عن بيوت العطلات.
- مراجعة كل مسميات حقول host/KYC. بعض المسميات الحالية صحيحة تقنياً لكنها تحتاج صياغة عربية أوضح قبل الإنتاج.
- تسجيل دخول الضيف يوجّه حالياً إلى `/dashboard`، وهذا خارج M9 ويرجع 404. نحتاج fallback مؤقت إلى أن توجد صفحة dashboard/profile الحقيقية.

## قاعدة التعامل

- إبقاء Phase 2 M9 مغلقاً.
- نقل أي ملاحظة من هذا الملف إلى خطة milestone لاحق فقط عند بدء تنفيذها.
- عدم خلط هذه الملاحظات مع مهام أمان auth/token/cookies.
